Assembly code:¶

Foto direct port manipulation:

Foto digitalWrite():

  • Over hoeveel clockcycles per seconde beschikt de Arduino Uno?
    De Arduino Uno heeft een kloksnelheid van 16 MHz, wat 16 miljoen clockcycles per seconde betekent.
  • Wat is de relatie tussen assembly code en clockcycles?
    Assembly code is een low-level programmeertaal die dus dicht bij de code voor de processor staat. Het aantal clockcycles dat nodig is om een bepaalde assembly instructie uit te voeren, hangt af van de instructie zelf en de architectuur van de processor.
  • Als je programmeert schrijf je geen assembly code. Wie genereert deze code en waarom?
    De compiler genereert deze code, zodat de programmeur in simpelere talen kan programmeren en de computer het nog steeds kan verstaan.
  • Assembly code instructies worden uitgevoerd door welk deel van de processor in onderstaande afbeelding?
    Assembly code instructies worden uitgevoerd door de ALU
  • Waarom resulteert dezelfde code niet in dezelfde assembly code als je je target verandert (en opnieuw compileert) van bv een Arduino Uno naar NodeMcu ESP8266?
    Dit komt doordat verschillende microcontrollers verschillende architecturen en instructiesets hebben. De compiler moet de code aanpassen aan de verschillende processors,wat dus zorgt voor aparte assembly code.
  • De code in het bestand firmware.lst is te lezen voor mensen en wordt ook wel …… genoemd.
    Assembly listing
  • Wordt de code in firmware.lst ook zo geïnterpreteerd door de processor? En zo nee; hoe dan wel?
    Nee, het wordt eerst omgezet in machinetaal, wat hierna door de processor uitgevoerd kan worden.
  • Waarom verandert er maar een klein beetje assembly code als je de code in je sketch verandert? Waarvoor dient de rest en waar komt het vandaan?
    Er veranderd dan maar een klein beetje, omdat het grootste gedeelte van de code bedoeld is om te zorgen dat je code goed runt, waarvoor alles goed afgesteld moet worden. Dit komt allemaal van de compiler.
  • Als ik een println() toevoeg komt er ineens veel meer in de listing te staan. Hoe komt dat?
    println() is een functie die extra code include zodat het overal werkt, op alle microcontrollers, maar ook bijvoorbeeld het formatteren ofzo.

Onderzoeksrapport Eli van Holland en Lars van Duijnhoven¶

Inleiding¶

Dit onderzoek is een schoolopdracht waarmee wij bekend worden met de gegeven microcontrollers door hier een literatuuronderzoek én een labonderzoek op uit te voeren.
Het literatuuronderzoek houdt in dat we van enkele microcrontrollers een lijst met specificaties hebben moeten maken. Dit betreft de volgende microcontrollers: Arduino Uno R3, ATTiny45, Lolin MCU V2 met ESP8266, ESP32, Raspberry Pi.
Het labonderzoek houdt in dat we de beroemde knipperende led code gaan schrijven, maar dan zo snel mogelijk laten knipperen en dit meten. Alle meetwaardes vergelijken we dan op het einde voor de hierboven genoemde controllers, op de ATTiny45 na.

Onderzoeksvraag¶

Deze onderzoeksvraag is gequote uit de opdracht.
"In dit onderzoek, dat uit verschillende delen bestaat, ga je antwoord geven op de hoofdvraag die luidt: “Wat zijn de voor een IoT-toepassing relevante eigenschappen van microcontrollers?”. Je gaat hierbij verschillende voorgeschreven microcontrollers/microcontrollermodules met elkaar vergelijken. De resultaten van het onderzoek kun je gebruiken tijdens de rest van deze course, maar zeker ook tijdens het project dat je aansluitend aan de coursefase gaat uitvoeren. Omdat dit een van de eerste onderzoeken in de course is en omdat de lijst van relevante eigenschappen van een microcontroller oneindig lang kan worden zijn deze eigenschappen beperkt tot de volgende lijst:

Vragen DNS en HTTP¶

  1. DNS berichten zijn berichten van de Dynamic Naming Service. Hiermee worden de URL's van websites omgezet in IP adressen, welke daarna weer gebruikt kunnen worden door laag 3 van het OSI model (de network layer).
  2. DNS werkt met beide, zowel TCP als UDP
  3. HTML wordt verstuurd met HTTP en HTTP gebruikt weer TCP, welke zich in de transport laag bevindt (laag 4).
  4. De webpagina wordt in zoveel stukken gestuurd, omdat hij anders over het limiet gaat en de andere mensen op het internet ineens een stuk minder kunnen versturen.
  5. Zodat de website sneller kan laden, omdat alle connecties het maar 1 keer hoeven te sturen en er dus niet gewacht hoeft te worden. Ik zou gokken dat het HTTPS er ook mee te maken heeft.
  1. Je kan deze legen door in de terminal sudo systemd-resolve --flushcaches uit te voeren.
  2. Ja, alleen is dit allemaal in hexadecimale getallen/ ASCII en is het dus niet goed leesbaar. Dit komt omdat de website informatie via TCP/ HTTP verstuurd wordt.

Ping van laptop naar PI¶

  1. Met het commando ping 192.168.1.11 (IP van de PI)
  2. Je kan je eigen IP adres van de laptop opvragen met ifconfig.
  3. Je kan het IP adres van je Raspberry PI opvragen door op de Raspberry PI ifconfig uit te voeren.
  4. De functie van ARP (Adress Resolution Protocol) is om het MAC-adres van de ontvanger te achterhalen.
  5. De functie van TTL (Time To Live) is dat oude packets verwijderd worden als ze te oud worden, zodat er geen onnodige packets blijven hangen.
  6. Het ethernet-adres van een broadcast is 1.1.1.1
  7. Ethernet is laag 1 en 2 van het OSI model.
  8. ICMP is laag 3 van het OSI model.
  9. Je kan de ARP cache verwijderen door ARP -d (IP adres) in te vullen in de terminal.

Opzetten van een putty-sessie naar PI¶

  5. Seq is bijvoorbeeld 1, voor een pakket van 49 bytes en als er dan gereageerd wordt met een ACK van dus 50, is de volgende sequence number 50.
  6. Zoals hiervoor uitgelegd is, is de ACK nummer dan 50 en de keer daarna, met een sequence number van 50 en een length van 25, wordt de ACK 75.
  7. Poort 48564 en 22 worden gebruikt hiervoor.

  9. Dan worden er een paar afsluitingsberichten via TCP en SSH gestuurd.
